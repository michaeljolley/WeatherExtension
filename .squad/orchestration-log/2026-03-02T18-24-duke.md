## Decisions Made
| Decision | Rationale |
|----------|-----------|
| Postal code-first UX | Reduces geocoding API flakiness; zip/postal codes are unambiguous |
| Keep city name fallback | International users may not know postal codes everywhere |
| Default: "Seattle" → "98101" | Consistency with postal-code-first guidance |
| Nominatim as postal fallback | Free, no API key, reliable postal → coordinates |
| Separate JSON for pinned locations | Keeps settings.json clean; supports complex data |
| One WrappedDockItem per pinned location | Matches SDK pattern; independent updates |
